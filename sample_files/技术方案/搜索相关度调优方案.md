# 搜索服务相关度调优方案

## 背景
当前搜索结果相关度偏低，用户反馈前 10 条命中率约 60%。
目标：将前 10 条命中率提升至 85% 以上。

## 现状
- 后端使用 Whoosh + jieba 做中文检索
- 目前仅使用 BM25 默认参数
- 字段权重未显式配置

## 方案

### 1. 字段权重调优
- name（文件名）：3.0 —— 文件名命中最相关
- path（路径）：2.0 —— 目录名是强信号
- tags（标签）：2.0 —— 运营打标签是强信号
- content（正文）：1.0 —— 默认权重
- code_index（代码索引）：1.0

### 2. 时间新鲜度加权
按文件修改时间做指数衰减：
`boost = 1.0 + 0.5 * exp(-age_days / 365)`
即最近修改的文件最高可获得 1.5 倍加权。

### 3. BM25F 参数
- K1 = 1.5（默认 1.2，稍放宽饱和）
- B = 0.75（全局）
- 字段级 B：name 0.5、path 0.6、content 0.75

### 4. 中文分词优化
- 引入 jieba 自定义词典（法律、财务、设计等行业术语）
- 对不可抗カ、违约金、劳动合同等词加权重
- 支持短语搜索（带引号）

## 评估
使用标注好的 500 个 query 做离线评测：
- MAP@10：目标 0.75+
- nDCG@10：目标 0.80+

上线后灰度一周，观察点击率与停留时长。
